# Agades PQC Gym Agent Environment

This Hugging Face Space skeleton exposes a public-safe Agent Environment for
AttackPlan JSON tasks. It lets users inspect task observations, score candidate
AttackPlans, and view rollout traces without publishing private training traces
or making cryptographic break claims.

Space bundle files:

- `hf/README.md`
- `hf/app.py`
- `hf/requirements.txt`
- `hf/dataset/attack_plans.jsonl`
- `hf/dataset/task_metadata.jsonl`
- `hf/dataset/rl_rollouts.jsonl`
- `hf/space_manifest.json`
- `hf/collection_manifest.json`

## Views

- Select any valid public toy/schema-only AttackPlan from the checked-in
  Hugging Face dataset bundle.
- Edit or paste AttackPlan JSON.
- Run the shared public verifier.
- Load a single-turn Agent Environment observation for the selected task.
- Score a candidate with the same pedagogical reward terms used by the public
  gym contract.
- Show `evaluation_status`, reward report, rollout trace, mock/real estimator
  status, and limitations.

## Packaging

`hf/requirements.txt` installs `agades-pqc-gym` from the public GitHub repo so a
standalone Space can import the shared verifier instead of relying on a local
`src/` checkout. The app reads the local dataset bundle first and falls back to
`examples/attack_plans/` for source-checkout development.

`hf/space_manifest.json` is generated by `agades-pqc hf-space-manifest`. It
records exposed example labels, the default example, dependency file, shared
verifier contract, Agent Environment contract, rollout trace schemas, and
no-code/no-security-claim guardrails expected before publishing the Space.
It also binds `hf/README.md`, the root Space README uploaded to the Hub, so the
Space carries `sdk: gradio`, `app_file: app.py`, and the `agent-environment`
tag in the repository metadata.

`hf/collection_manifest.json` is generated by `agades-pqc hf-collection-manifest`.
It records the reviewed Hugging Face Collection grouping for the GitHub repo,
dataset, Space, benchmark card, source catalog, and public benchmark manifest.

## Hub Publication

Use a private Space first and keep public release behind review:

```bash
hf auth login
hf repos create AgadesTech/agades-pqc-gym-agent-env --type=space --space-sdk gradio --private --exist-ok
hf upload AgadesTech/agades-pqc-gym-agent-env hf . --repo-type=space --commit-message "Sync Agades PQC Gym Agent Environment"
```

For automation, prefer `HF_TOKEN` rather than passing tokens on the command
line. Publish publicly only after release review confirms the dataset bundle,
Space manifest, and no-claim guardrails are current.

## Guardrails

- Do not execute arbitrary code.
- Do not accept live third-party targets.
- Do not claim any real-world PQC break.
- Do not publish private RL traces, reviewer annotations, prompts, model
  weights, adapters, or serious-research rollouts.
- Clearly mark unsupported families and mock outputs.
