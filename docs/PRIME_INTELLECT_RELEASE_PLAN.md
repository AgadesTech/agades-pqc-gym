## Positioning

Agades PQC Gym should be framed as a deterministic AttackPlan verifier environment:

```text
agent submits AttackPlan JSON
  -> verifier validates and routes by family
  -> evaluator returns score or unsupported status
  -> run ledger records candidate, metrics, warnings, and trace pointer
```

The Prime-facing story should cite
`https://app.primeintellect.ai/dashboard/home/quickstart` as the onboarding
surface, `https://www.primeintellect.ai/auto-nanogpt` as the auto-nanoGPT public
benchmark story, and the public
`PrimeIntellect-ai/experiments-autonomous-speedrunning` repository as an
observability and evaluator-led-run pattern. These are source-map-only
ecosystem anchors for the current MVP; Agades submissions stay JSON-only and are
not a security claim, Prime RL training run, or published private evolution
trace.

## Current Prime Quickstart Alignment

As of 2026-05-18, the public Prime quickstart onboarding path is recorded in
`docs/prime_publication_handoff.json` for review reproducibility:

```bash
uv tool install -U prime
prime login
prime lab setup
prime eval run primeintellect/reverse-text -m openai/gpt-oss-20b -p prime -n 1 -r 1 -t 512 -s -A
prime eval run primeintellect/aime2026 -m openai/gpt-oss-20b -p prime -n 1 -r 1 -t 2048 -s -A
```

After private Prime Hub review and owner-qualified publication, the Agades
environment should be exercised through:

```bash
prime eval run <owner>/agades-pqc-verifier-env
```

These commands are source-alignment references only: external Prime execution has not been performed, Prime CLI credentials are not checked at generation time, and hosted compute or public visibility still requires explicit release review.

## Prime Autonomy Harness Alignment

The Prime AutoNanoGPT story published on 2026-05-14 describes an autonomous
research harness with `AGENTS.md`, `goal.md`, `plan.md`,
`scratchpad/THREAD.md`, run logs, scripts, and configs. Agades maps that pattern
to public, reviewable repository artifacts instead of publishing private
scratchpads:

- `AGENTS.md`: repository-level safety and code-quality rules.
- `docs/PLAN.md`: stable milestone plan.
- `docs/IMPLEMENT.md`: reproducible command runbook.
- `docs/STATUS.md`: durable long-running implementation log.
- `public/run_export/manifest.json`: public run observability export.
- `docs/private_run_policy.json`: private moat and release boundary.

This mapping is recorded in `docs/prime_speedrun_handoff.json` under
`autonomy_harness_alignment`. It is a source-alignment contract only. The exact
guard is: external Prime autonomous run has not been performed. Private
scratchpads and private evolution traces are not published, and Prime-hosted
autonomy still requires credentialed review.

Prime Hub publication requires credentials and release review before any
external upload.

Before any Prime Hub push, regenerate and verify the private-run policy and
local publication preflight, plus the Prime publication handoff:

```bash
uv run agades-pqc prime-publication-handoff --out docs/prime_publication_handoff.json
uv run agades-pqc prime-publication-handoff-verify --handoff docs/prime_publication_handoff.json
uv run agades-pqc prime-speedrun-handoff --out docs/prime_speedrun_handoff.json
uv run agades-pqc prime-speedrun-handoff-verify --handoff docs/prime_speedrun_handoff.json
uv run agades-pqc private-run-policy --out docs/private_run_policy.json
uv run agades-pqc private-run-policy-verify --policy docs/private_run_policy.json
uv run agades-pqc publication-preflight --out public/publication_preflight.json
uv run agades-pqc publication-preflight-verify --preflight public/publication_preflight.json
uv run agades-pqc external-publication-review-packet --out docs/external_publication_review_packet.json
uv run agades-pqc external-publication-review-packet-verify --packet docs/external_publication_review_packet.json
```

The committed `docs/private_run_policy.json` and
`public/publication_preflight.json`, summarized by
`docs/external_publication_review_packet.json`, keep Prime publication blocked
from private traces, prompt policies, evaluator weights, unreleased candidates, and
responsible-disclosure material until release review and credential review are
explicitly approved.
