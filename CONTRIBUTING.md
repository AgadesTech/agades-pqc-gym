# Contributing

Contributions should improve schemas, validators, estimator boundaries, documentation, or reproducible toy benchmarks.

## Rules

- Keep all examples on toy, downscaled, public, or explicitly authorized targets.
- Do not submit exploit chains against live systems.
- Do not add a fake estimator for an unsupported family.
- Mark mock outputs clearly.
- Include tests for behavior changes.

## Useful Contributions

- Family adapter skeleton improvements.
- Better validation findings.
- Lattice Estimator wrapper tests against pinned public baselines.
- Hugging Face or Prime Intellect verifier packaging.
- Documentation clarifying assumptions and limitations.
